# B2B Game Idea Generation for LinkedIn

MVP на n8n, управляемый через Telegram-бота, для генерации B2B-идей для LinkedIn-постов, текстов постов и визуальных концепций для студии в сфере game development / outsourcing / co-development.

Проект был реализован как тестовое задание и демонстрирует рабочий automation flow: от генерации темы до утверждения готового поста.

## Что делает бот

Telegram-бот ведет пользователя по полному сценарию подготовки контента:

1. Генерирует 12 тем для LinkedIn-поста:
   - 6 тем на основе LinkedIn-релевантных сигналов
   - 6 тем на основе интернет-источников / индустриальных материалов
2. Позволяет выбрать тему.
3. Спрашивает язык поста: English или Russian.
4. Генерирует LinkedIn-пост для B2B-аудитории игровой индустрии.
5. Генерирует 3 варианта изображения для поста.
6. Позволяет выбрать изображение или сгенерировать новые варианты.
7. Показывает финальный предпросмотр.
8. Принимает комментарии для правки текста или изображения.
9. Сохраняет финальный результат как утвержденный пост.

В текущем MVP кнопка **Publish / Публиковать** сохраняет финальный результат как утвержденный и готовый к публикации. Реальную автопубликацию в LinkedIn можно добавить отдельным публикационным слоем.

## Архитектура

```text
Telegram Bot
→ n8n workflow
→ PostgreSQL
→ LLM API
→ Image Generation API
→ Server file storage
→ RSS / trusted sources / LinkedIn-related source corpus
````

Workflow сделан stateful: состояние пользователя, выбранная тема, выбранный язык, версии постов, партии изображений, правки и утвержденные результаты сохраняются в PostgreSQL.

## Карта состояний

```text
idle
→ generating_topics
→ choosing_topic
→ choosing_language
→ generating_post
→ choosing_image
→ preview
→ revising_text
→ revising_image
→ approved
```

Такая state machine помогает не терять контекст, когда пользователь просит новые темы, новые изображения или отправляет комментарии для правок.

## Логика источников

### Интернет-источники

В MVP используются RSS / trusted sources. Материалы фильтруются по ключевым словам, связанным с игровой индустрией и B2B-услугами:

* game production
* co-development
* outsourcing
* live ops
* QA / testing
* art production
* Unity / Unreal
* production risk
* cost / quality / timelines

LLM преобразует эти сигналы в практические B2B-темы для LinkedIn, а не просто пересказывает новости.

### LinkedIn-релевантные сигналы

Прямой масштабный парсинг LinkedIn чувствителен к правилам платформы, поэтому в MVP используется более безопасный подход:

* заранее заданный список релевантных публичных LinkedIn-страниц компаний;
* периодический refresh-процесс;
* очистка текста и сохранение chunks в PostgreSQL;
* ротация источников и балансировка по компаниям / категориям.

Основной Telegram workflow не парсит LinkedIn во время пользовательского запроса. Он использует уже подготовленные chunks из базы. Это делает работу бота быстрее и стабильнее.

Для больших корпусов источников архитектуру можно расширить через embeddings и RAG.

## Как импортировать workflow

1. Открыть n8n.
2. Создать новый workflow.
3. Импортировать `workflows/B2B_game_idea_generation_linkedin.n8n.json`.
4. Заново подключить credentials:

   * Telegram Bot API
   * PostgreSQL
   * OpenAI / LLM API
   * Image Generation API
5. Обновить пути и URL под свое окружение.
6. Активировать Telegram Trigger.

Экспортированный workflow очищен. Credentials и токены в файл не включены.

## Публикация в LinkedIn

Реальная публикация в LinkedIn в текущий MVP workflow не встроена.

Архитектура позволяет добавить ее позже одним из способов:

1. Прямая интеграция с LinkedIn API:

   * LinkedIn Developer App
   * OAuth 2.0
   * `w_member_social` для публикации в личный профиль
   * `w_organization_social` для публикации на страницу компании
   * Posts API
   * Images API для постов с изображениями

2. Сторонний publishing API:

   * Post for Me
   * PublishQ
   * Buffer / аналогичные publishing-платформы

В этом случае текущая таблица `approved_posts` может использоваться как очередь публикаций. Отдельный publishing workflow сможет брать утвержденные посты и отправлять их в LinkedIn.

## Промпты

В workflow используются отдельные промпты для:

* генерации тем;
* генерации LinkedIn-поста;
* генерации визуальных концепций;
* правки текста;
* правки изображения.

Промпты возвращают структурированный JSON. Это упрощает парсинг результата и делает workflow стабильнее в n8n.

## Ограничения

* MVP демонстрирует логику workflow и Telegram UX.
* Качество финального текста и изображений зависит от выбранных моделей и промптов.
* Автопубликация в LinkedIn требует отдельной интеграции.
* Для production-версии нужно добавить более сильные retry-ветки, мониторинг и semantic duplicate detection.
* Для больших корпусов источников рекомендуется использовать RAG.
